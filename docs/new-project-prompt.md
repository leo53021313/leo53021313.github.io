# 新作品 intake prompt（完成新專案後，填完整段貼回給 Claude）

> **用法**：每做完一個作品，把下面 `===` 之間整段複製、填好（**中文寫就好，英文版我幫你翻並標出要校稿的技術詞**），連同截圖一起貼回給我。我會照 [`ADD-PROJECT.md`](../ADD-PROJECT.md) + `src/content.config.ts` 的 schema，幫你新增/更新**中英雙語**作品頁。
>
> 規則：**沒有的就留白或寫「無」——我不會編造數據**。量化成果這段最重要，盡量給真實數字（去 `COUNT(*)`、跑一次 eval、看 dashboard 量一下）。

```text
=== 新作品 intake（填完貼回給 Claude）===

【給 Claude 的指令】
依下列資訊，照 ADD-PROJECT.md + content.config.ts schema 新增/更新作品：
- 中英雙語、共用 CaseStudy 元件、照 _template.md 的四段骨架（問題/做法/架構/踩坑）。
- metrics 只用我給的真實數字；缺的提醒我補，不要編造。
- 英文版技術名詞照業界慣例翻；不確定處標註讓我校稿。
- 「架構」我給箭頭流程，請你轉成 Mermaid 圖。
- featured / order 你依強度建議；最強證據（有 live demo 的）排前面。
- 若有填【接案視角】，敘事中順帶帶出商業價值與對應的服務（誠實、不誇大）。
- 完成後跑 npm run build + npm test，並回報 zh/en 對等檢查有沒有過。

【基本資料】
- slug（網址用，英數+連字號，例 my-cool-project）：
- 標題：
- 一句話描述（oneLiner，顯示在標題下）：
- 我的角色（role，個人/團隊・我負責什麼）：
- 狀態：done / wip
- 時間（timeframe，例 2026-04 ~ 至今）：
- 技術棧（tech，逗號分隔：語言/框架/服務/DB）：
- 標籤（tags，2–4 個領域詞，當 kicker chips，例 資料工程/爬蟲/LLM）：

【證據】（有什麼填什麼）
- Live 網址（demoUrl，只放真的打得開的）：
- Repo 網址（repoUrl，公開才放；private 留白 → 自動顯示「程式碼可應要求提供」）：
- 封面截圖（cover）：附圖檔 ⤵；一句話描述截圖內容（coverAlt）：
- 其他證據（evidence，無 live demo 時：截圖/GIF 在呈現什麼）：

【量化成果 ★最重要】（每條＝一個數字＋單位，給 3–6 條）
範例方向：
  資料規模：累積 N 筆 / N 個 SKU / N 個來源 / 涵蓋 N 類
  運作規模：每日 N 次 / 每次處理 N 筆 / 已運行 N 天
  成效：準確率 N% / 解題率 N/100 / 省時 N% / 成本 $N/月 / uptime N%
※ 至少 1 條用「價值」講（省時/省成本/規模/可靠度），別全是技術指標 ← 接案方最看這個。
※ 怎麼拿數字：DB 跑 SELECT COUNT(*)；起訖時間用 `git log --reverse --format=%ad`；
   ML 跑一次 eval 算成功率；排程/流量看 dashboard 或 log。真沒有就寫「無」。

【敘事】（四段，每段 2–4 句白話，我幫你潤稿+翻譯）
- 問題：要解決什麼？為什麼值得做？
- 做法：怎麼做、為什麼選這些技術？
- 架構：資料/流程怎麼走（給箭頭，例：爬蟲 → 清洗 → DB → API → 前端）：
- 踩坑：遇到什麼坑、怎麼解、學到什麼？（誠實，這段對技術主管最加分；別留「待補」）

【補充】（沒有就留白）
- 相關連結（技術文章 / Demo 影片 / 投影片 / 得獎 / 黑客松）：
- 想強調的一句亮點：

【接案視角・選填】（偏接案/合作用；沒有就留白）
- 這個作品對應到我哪個服務？（資料爬取/清洗管線・自動化排程+告警・LLM 應用整合・資料視覺化網頁，或自己寫一句）：
- 若是真實客戶案：產業/情境（可匿名，例：某電商）＋我交付了什麼＋結果：
- 可放的客戶好評/回饋（一句即可，需對方同意）：
```

## 為什麼問這些（對應招募方在看的東西）

研究多個中英文工程師作品集指南後，跨來源一致看重的就這幾項，這份表單逐項對應：

| intake 欄位 | 招募方在乎的 | 對應 schema / 區塊 |
|---|---|---|
| 問題 | 真實問題、為何值得做 | body `## 問題` |
| 做法 + 技術棧 | 技術選擇與判斷 | body `## 做法` + `tech` → 技術棧區塊 |
| 架構（→Mermaid） | 端到端流程、系統設計 | body `## 架構` |
| **量化成果** | **最被強調**——數字證明影響力 | `metrics` |
| 踩坑 | 解決問題的能力、誠實反思 | body `## 踩坑` |
| 證據（demo/截圖/repo） | 看得到、跑得起來 | `demoUrl`/`cover`/`evidence`/`repoUrl` |
| 角色 | 你實際做了什麼 | `role` |
| 補充連結 | 技術部落格/開源/得獎加分 | （目前放 body；常用的話我再加 `links` 欄位） |

> 來源：104 職場力、Yourator、Cake、ALPHA Camp（中文）；InterviewMaster 2025 DS Portfolio Guide、TripleTen、data-science README templates（英文）。
